# GRM 주간 퀴즈 v1 운영설계

- 작성일: 2026-07-18
- 범위: 문항 데이터와 주간 운영 절차
- 비범위: 웹 화면 구현, 참여 이력 저장, 보상 기능

## 1. 목적과 운영 기준

GRM 주간 퀴즈는 전 직원이 규제·품질 용어와 최근 공개 사례를 짧게 복습하도록 돕는 학습 기능이다. 지식 경쟁이나 인사평가가 목적이 아니며, 모든 정답은 GRM의 공개 근거에서 즉시 재확인할 수 있어야 한다.

정본 문항 뱅크는 `web/data/quiz_bank.json`이다. v1 뱅크는 28문항이며 난이도는 `easy` 17문항(60.7%), `normal` 11문항(39.3%)으로 구성한다. 출처 구성은 `glossary` 17문항, `brief` 8문항, `finding` 3문항이다.

## 2. v1 UX 원칙

### 2.1 랭킹 없음

- 개인·부서·직급 간 순위표를 만들지 않는다.
- 점수로 경쟁을 유도하거나 인사평가 자료로 사용하지 않는다.
- v1의 성공 기준은 정답률 경쟁이 아니라 짧은 복습의 완주와 근거 확인이다.

### 2.2 정답 즉시 확인

- 선택 직후 정답·오답 여부와 `explanation_ko`를 같은 화면에서 보여준다.
- 해설과 함께 `source_ref`를 제공해 사용자가 근거 용어 또는 공개 카드·finding으로 바로 이동할 수 있게 한다.
- 오답 후에도 정답 선택지를 숨기지 않으며, 재시도 강제나 감점은 두지 않는다.
- 근거 링크가 열리지 않으면 답안을 확정적으로 노출하지 않고 콘텐츠 오류 상태로 처리한다.

### 2.3 주 3~5문항 노출

- 기본 노출량은 주 4문항으로 하고 운영자가 3~5문항 범위에서 조정할 수 있게 한다.
- 한 주 세트에는 `easy`를 과반으로 포함하고 `normal`은 1~2문항으로 제한한다.
- 같은 주에는 동일 `source_ref`를 원칙적으로 한 번만 사용한다. 하나의 카드나 finding에서 여러 문항이 필요하면 서로 다른 명시 사실을 묻는지 검증 게이트에서 확인한다.

### 2.4 전 직원 대상 난이도

- `easy`는 약어·용어·GRM 화면의 기본 개념을 묻고, 비전문 직원도 용어집만 읽으면 답할 수 있게 한다.
- `normal`은 브리프·findings에 명시된 업체, 조치, 지적사항 또는 회수 사유를 묻되 별도 법령 지식을 요구하지 않는다.
- 규제 조항·수치·기간은 근거에 그대로 있을 때만 사용한다. 자료 밖 상식이나 기억을 정답 근거로 보강하지 않는다.
- 부정형·함정형 질문을 피하고, 오답 선택지는 독립된 실제 개념·업체·품질 이슈를 사용하되 사실을 단정적으로 왜곡하지 않는다.

### 2.5 접근성과 피드백

- 선택지는 키보드로 탐색·선택할 수 있고 정답 상태는 색상 외에 텍스트와 아이콘으로도 전달한다.
- 정답 확인 전에는 선택지를 한 번 변경할 수 있게 하고, 확인 후에는 근거를 읽을 충분한 시간을 보장한다.
- 모바일에서도 질문, 네 선택지, 정답 해설과 근거 링크가 한 흐름으로 읽히게 한다.

## 3. 주간 자동 생성 흐름 제안

예약 세션은 최근 발행본이 공개된 뒤 실행한다. 예시는 매주 화요일 09:00 KST이며, 실제 시각은 주간 브리프의 공개 완료 시점에 맞춰 조정한다. 공개 전 브랜치의 카드 URL은 사용하지 않는다.

### 3.1 입력 고정

1. 예약 세션 시작 시 `origin/main`을 fetch하고 clean worktree를 확인한다.
2. 허용 입력을 `web/data/glossary.json`, `web/data/briefs/*.json`, `https://grm-solutions.com/findings/`의 공개 finding으로 제한한다.
3. 직전 기출 이력에서 문항 ID, 정규화 질문문, 정답, `source_ref`, 근거 사실 지문(fact signature)을 불러온다.
4. 세션 시작 시각과 입력 커밋 SHA를 실행 로그에 남겨 이후 검증을 재현할 수 있게 한다.

### 3.2 후보 생성

1. 먼저 노출 목표 3~5문항의 2~3배수 후보를 만든다.
2. 생성 단계는 근거 텍스트를 벗어나지 않고 질문, 네 선택지, 정답, 1~2문장 해설을 작성한다.
3. `easy`와 `normal` 비율, 출처 유형, 최근 노출 이력을 고려해 후보를 섞는다.
4. 생성 모델의 출력은 초안으로만 취급하며 검증을 통과하기 전에는 공개 산출물에 반영하지 않는다.

### 3.3 검증 게이트

다음 검사를 모두 통과해야 주간 세트를 확정한다.

1. **구조 게이트:** UTF-8, JSON 파싱, 필수 키, 선택지 정확히 4개, `answer_index` 0~3, 허용 enum, ID 고유성을 검사한다.
2. **근거 게이트:** `glossary`는 ID와 정의 본문을, `brief`는 저장소 카드 ID·표현과 공개 딥링크 앵커를, `finding`은 공개 단건 조회 결과와 딥링크를 대조한다.
3. **사실 게이트:** 정답과 해설의 모든 고유명사·수치·기간·조항이 근거에 명시되어 있는지 상위 검증 모델이 문항별로 판정한다. 하나라도 근거 밖이면 폐기한다.
4. **오답 게이트:** 오답이 정답이 될 여지가 없는지, 선택지 자체가 허위 사실을 단정하지 않는지, 복수정답 가능성이 없는지 확인한다.
5. **접근성 게이트:** 모든 URL이 열리고 brief 앵커 또는 finding ID가 실제로 존재하는지 확인한다.
6. **노출 게이트:** 주 3~5문항, easy 과반, normal 1~2문항, 동일 `source_ref` 주 1회 원칙을 확인한다.

어느 게이트든 실패하면 자동 게시하지 않는다. 실패 후보를 제외하고 남은 후보로 다시 구성하며, 최소 3문항을 확보하지 못하면 해당 주 세트를 보류하고 운영자에게 원인과 실패 문항 ID를 알린다.

### 3.4 기출 중복 방지

- **완전 중복:** 질문문을 공백·문장부호·대소문자 기준으로 정규화한 해시가 과거와 같으면 제외한다.
- **근거 중복:** `source_ref + fact signature`가 같으면 표현만 바꾼 재출제로 보고 기본 12주 동안 제외한다.
- **유사 중복:** 질문문과 정답의 의미 유사도가 기준을 넘으면 자동 확정하지 않고 검토 대상으로 보낸다.
- **용어 재등장:** 전 직원 핵심 용어는 12주 뒤 재출제를 허용하되 질문 관점과 오답 구성을 바꾸고, 누적 오답률 같은 개인 데이터는 v1에서 사용하지 않는다.
- 중복 방지 이력은 주간 세트가 확정된 뒤에만 추가한다. 생성 실패나 미공개 후보는 기출로 기록하지 않는다.

### 3.5 주간 산출과 감사

- 확정 세트에는 생성 주차, 입력 커밋 SHA, 선택된 문항 ID, 검증 시각과 검증 결과를 기록한다.
- 웹 구현 단계에서는 문항 본문을 복제하지 않고 정본 뱅크의 ID를 참조해 주간 세트를 구성한다.
- 게시 직전 링크를 한 번 더 확인하고, 게시 후에는 화면에서 정답·해설·근거 링크가 일치하는지 스모크 테스트한다.
- 출처 내용이 수정·비공개되거나 링크가 사라지면 해당 문항을 비활성화하고 대체 문항을 선택한다.

## 4. v1 운영 역할

- **예약 세션:** 후보 생성, 기계 검증, 중복 검사, 주간 세트 제안
- **상위 검증 모델:** 근거 대조, 정답 유일성, 오답의 비왜곡성, 해설의 자료 내 한정 여부 전수 판정
- **운영자:** 검증 실패 예외와 링크 소실 확인, 최종 공개 승인
- **웹 애플리케이션:** 선택·즉시 피드백·근거 이동만 담당하며 문항 사실을 재작성하지 않음

## 5. 보류 사항

다음 항목은 v1에 포함하지 않으며 별도 개인정보·운영정책과 데이터 모델을 합의한 뒤 검토한다.

- 개인별 참여 여부, 응답 이력, 정답률과 오답 문항 저장
- 연속 참여(streak), 뱃지, 레벨, 포인트와 보상
- 개인·부서별 맞춤 난이도와 추천 문항
- 관리자용 문항 편집·승인 UI와 세부 분석 대시보드
- 개인별 알림, 미참여자 재촉, 이메일·메신저 리마인더
- 인사 시스템 또는 교육 이수 시스템과의 연동

랭킹은 단순 보류 기능이 아니라 v1 원칙상 비목표다. 향후에도 도입하려면 학습 목적, 심리적 안전, 개인정보 최소수집과 인사평가 비연계 원칙을 먼저 재검토해야 한다.

## 6. v1 완료 기준

- 주간 3~5문항을 정본 ID로 선택할 수 있다.
- 각 문항은 선택 직후 정답, 1~2문장 해설과 근거를 보여줄 수 있다.
- 모든 `source_ref`가 허용 근거에 연결되고 자동 검증에서 실재함이 확인된다.
- 기출 완전 중복과 동일 근거·동일 사실의 표현만 바꾼 재출제를 차단한다.
- 참여 기록·뱃지·랭킹 없이도 전 직원이 부담 없이 완주할 수 있다.
