# GRM 퀴즈 운영설계 addendum — week 회전·성장 시스템

- 작성일: 2026-07-18
- 대상 정본: `web/data/quiz_bank.json`
- 상위 문서: `docs/specs/GRM_퀴즈_운영설계_2026-07-18.md`
- 예약 세션 정본: `docs/prompts/GRM_주간퀴즈생성_프롬프트_v1.md`

## 1. 목적

초기 고정 문항 뱅크를 매주 라이브 브리프와 함께 성장시키되, 같은 주의 세트가 중간에 바뀌거나 과거 출제 이력이 덮어써지지 않게 한다. `week`는 생성 이력과 주차 소유권의 정본 키이며 형식은 KST 실행일 기준 ISO 주차 `YYYYWW`다.

## 2. 데이터 계약

- 기존 시드 문항은 `week`가 없을 수 있다. 이는 상시 회전 가능한 legacy pool이다.
- 예약 세션이 추가하는 모든 문항에는 `week`가 필수다. 한 주에는 3~5문항(기본 4문항)을 같은 `week`로 묶는다.
- ID는 `q-{YYYYWW}-{NN}`이고 전역 고유하다. 공개 후 `id`, `week`, 질문, 선택지 순서, 정답과 근거는 불변이다.
- 문항은 append-only다. 사실 오류나 링크 소실로 사용을 중단해야 하면 과거 기록을 조용히 고치지 말고 후속 비활성화 스키마를 먼저 설계한다.
- 같은 `week`가 이미 하나라도 있으면 예약 세션은 재실행하지 않는다. 부분 생성 복구는 자동 세션이 임의 판단하지 않고 별도 수리 작업으로 처리한다.

## 3. 주차 선택 규칙

주간 소비자는 다음 우선순위를 따른다.

1. 현재 `YYYYWW`와 정확히 일치하는 문항이 3~5개 있으면 그 명시 세트를 그대로 노출한다.
2. 현재 주차 문항이 없으면 `week` 없는 legacy pool에서 ISO 주차 seed로 결정론 선택한다.
3. 현재 주차 문항이 1~2개뿐이거나 6개 이상이면 손상된 명시 세트로 보고 자동 혼합하지 않는다. legacy fallback과 운영 경보 중 하나를 제품 정책으로 명시해야 하며, 조용히 일부만 노출하지 않는다.

현재 `web/assets/quiz.js`는 전체 뱅크를 난이도별로 나눈 뒤 ISO 주차 seed로 회전하며 `week`를 아직 소비하지 않는다. 따라서 이 addendum의 1순위 명시 세트 보장은 후속 소비자 구현 사항이다. 그 전까지 `week`는 중복 생성 방지·감사 메타데이터로만 작동하며, 신규 문항이 생성 주에 반드시 화면 선택된다고 가정하지 않는다.

## 4. 성장과 안정성 규칙

- 월요일 13:00 생성·머지는 그 주 라이브 브리프가 확정된 뒤 수행한다. 라이브보다 앞선 브랜치의 카드 앵커를 예약하지 않는다.
- 명시 주차 세트를 소비하도록 전환한 뒤에는 뱅크가 커져도 과거·현재의 명시 세트가 바뀌지 않는다. legacy seed 회전만 뱅크 길이에 영향을 받을 수 있다.
- 한 주의 문항은 서로 다른 카드를 우선하고, 같은 `source_ref`는 원칙적으로 한 번만 쓴다.
- 최근 12주에 같은 `source_ref`와 같은 사실을 물은 문항은 다시 만들지 않는다. 핵심 용어 재출제는 12주 뒤에도 질문 관점과 오답 구성이 달라야 한다.
- 난이도는 주 세트에서 `easy` 과반, `normal` 1~2를 유지한다. 누적 뱅크 전체 비율보다 각 주 학습 부담을 우선한다.
- 성장량은 주당 최대 5문항으로 제한한다. 후보가 많아도 근거 명시성·정답 유일성·오답 비왜곡을 우선한다.

## 5. 게이트와 감사

`quiz_lint.py`는 스키마, ID, 선택지 수, 정답 인덱스, 난이도, 실제 ISO `week`, 로컬 glossary/brief 참조, 외부 URL 형식과 내부 개념 금지를 검사한다. 의미 중복, 정답 유일성, 카드 사실과 해설의 문장 단위 정합성, 오답 왜곡은 생성 세션이 별도로 전수 확인한다.

주차 확정 기록에는 최소한 다음을 남긴다.

- `week`, 생성 문항 ID와 근거 카드 ID
- 입력 `origin/main` SHA와 최신 브리프 `publish_date`
- `quiz_lint.py` 결과와 전체 테스트 명령
- PR URL, CI 결과, merge SHA 또는 `SKIP` 사유

검증 실패 주는 빈 주차로 남을 수 있다. 지난 문항을 복제해 연속성을 꾸미거나 lint 규칙을 완화하지 않는다.

## 6. 후속 소비자 전환 완료 기준

- `quiz.js` 또는 대체 소비자가 현재 `week`의 3~5문항을 우선 선택한다.
- 명시 세트 부재 시에만 legacy 결정론 회전으로 폴백한다.
- 뱅크 append 전후 같은 명시 주차의 선택 결과가 byte-equivalent임을 테스트한다.
- 부분·과다 주차 세트가 조용히 노출되지 않는 fail-loud 테스트가 있다.
- ISO 연도 경계(1주차·52/53주차)와 KST 기준 계산을 고정 테스트한다.
