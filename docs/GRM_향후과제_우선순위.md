## 0. 우선순위 원칙

현 시스템은 **고장 대기 상태가 아니라 정상 운영 + 검증창이 열린 상태**다. 따라서:

1. **지금 열린 관찰을 *객관적으로 종료*시키는 작업이 최우선** (관찰을 수동 대기가 아니라 능동 종료로).
2. **사용자 수와 무관하게 신뢰도를 갉아먹는 침묵 실패가 그다음** (단일 사용자여도 신호 누락은 사고).
3. **다사용자·누적데이터·신규 소스처럼 관찰 결과나 상용화에 종속된 것은 뒤로** (지금 정하면 근거 없는 결정).

> 핵심: 지금은 새 기능을 더하기보다 **열린 관찰을 닫는 데 집중**하고, 그걸 지키는 침묵-실패 방어를 붙인 뒤, 확장은 관찰 결론이 난 다음 결정한다.

## P0 — 지금 (열린 관찰을 객관화/방어)

### P0-1. 발행물 품질 Eval 하니스 〔EVAL-1〕 — 착수
- **문제**: K3 종료조건 "4주 연속 품질 유지"가 정성적. 품질 판정을 사람이 눈으로(그것도 "대충") 하고 있어 결함을 놓치고 관찰이 막연한 대기가 됨.
- **해결**: 발행 후 독립 세션의 AI 채점자가 사실정합·차별화·시사점/점검 품질을 **정량 채점**(E1~E6). **다관점 페르소나** 채점으로 단일 운영자가 스킵하던 디테일을 표면화.
- **현황**: 설계문서 + 실행프롬프트 v0 작성 완료(`GRM_Brief_Eval_harness_design.md`·`prompts/GRM_Brief_Eval_실행프롬프트.md`). 이번 주 발행분부터 시범 채점 가능.
- **의존성**: 없음(비파괴, scaffold/golden/v16 불변). · **난이도**: 낮음(E1 프롬프트)~중(E2 Python).
- **효과**: K3 관찰을 객관화·가속. "막연한 대기" → "스코어카드 기반 능동 종료".

### P0-2. GAP-2 모달리티 근본해결 (브랜드명만 있는 생물주사제 오분류) — 🟡 Codex GO·머지 대기
- **문제**: 자닥신주·Hizentra 패턴 — 원료/클래스 텍스트 없이 브랜드명만 있으면 Biologic 이 Chemical 로 오분류. (제형 접미사 `주`가 1순위 생물 판정을 못 넘긴 항목을 2순위에서 Chemical 로 확정)
- **왜 P0**: BIO-1 관찰의 *전제*. 바이오 칸 누적을 보고 BIO-2 착수를 판단하는데 모달리티가 틀리면 그 판단 근거가 오염.
- **해결(채택)**: **큐레이티드 사전 `MODALITY_BIOLOGIC_BRANDS`**(근거 있는 2개: 자닥신·hizentra)를 `compute_modality()` 1순위 생물 블록(제형접미사 앞)에 부분문자열 매칭으로 삽입. 좁은 범위 — MFDS 제품허가 API(15095677, 주성분 정형필드)는 PL-7 별도 트랙으로 미룸(사용자 승인 2026-06-11).
- **결과**: 회귀 GAP2-1~5 + 의도적 supersession 3건(HC 상세fetch 실패 graceful→Chemical 을 사전 적중분 Biologic 일원화), 341 green·golden/scaffold/v16 불변. Codex GO(P-1·P-2 추인). branch `fix/gap2-brand-biologic` `5840b3d`. 비차단 P3=제품명 키 합산 하드닝 후속.
- **의존성**: 없음. · **난이도**: 중 → 실제 낮음(좁은 사전). · **산출물**: `archive/point-in-time/GRM_GAP2_modality_ClaudeCode_지시.md`·`…_Codex_review_prompt.md`. P0-1 의 E1-c·바이오 페르소나가 잔여 brand-only 오분류를 계속 *드러내는* 관측 짝.

## P1 — 침묵 실패 방어 (사용자 수 무관, 신뢰도 직결)

### P1-1. MON-P2 상태저장형 관측성 (Codex heartbeat 요약)
- **문제**: 현 P1 health 는 그날 그 run 만 보는 stateless → "연속 N일 0건·소스 침묵 추세"를 아무도 못 잡음. WHO NOC 침묵으로 배치4에서 한 번 데임.
- **해결**: 최근 7일 Actions/Issue/health 요약 + 소스별 baseline 대비 이탈(연속 0건) 감지. health JSON 누적 또는 Notion 이력 쿼리.
- **왜 P1**: 관찰 중 침묵 누락이 생기면 관찰 데이터 자체가 조용히 망가짐 → 관찰을 *지키는* 장치.
- **의존성**: 없음(기존 §5.3 P2 트랙). · **난이도**: 중. · **현황**: 미착수.

### P1-2. PL-10b / B1 멱등성 근본해결
- **문제**: Status 큐를 날짜 윈도우로 조회하는 구조 → Status 갱신 실패 시 재유입(PL-10b) 또는 미소비 New 누락(B1, 거울상).
- **현황**: 임시방어 적용됨(조회윈도우 30일 + `aged-unconsumed-new` 경고). 단일 사용자 단계에선 급한 불은 꺼짐.
- **순서**: **K3 관찰 종료 직후 착수**. 근본해결(날짜 하한 제거 + handoff 생성 시 row "handed-off" 표시)은 임시방어를 무해하게 흡수. 트리거: 주간 Routine 잦은 지연 / 상용화 직전이면 앞당김.
- **의존성**: K3 관찰 종료 권장. · **난이도**: 중~상(수집기측 Status 전이 결합).
