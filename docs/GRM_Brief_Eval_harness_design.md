# GRM Brief Eval 하니스 — 설계문서 (EVAL-1)

> **한 줄 정의.** 발행된 Weekly Brief 의 **내용 품질**(사실 정합·카드 차별화·시사점/점검 유용성)을 매주 **독립 세션의 AI 채점자**가 정량 점수로 평가하는 게이트. 사람이 눈으로 "대충 읽는" 관찰을 **객관 점수**로 대체해 K3 4주 관찰의 종료 판정을 빠르고 신뢰성 있게 만든다.

## 1. 왜 필요한가 (문제 정의)

### 1.1 현재 품질 통제의 공백

GRM 의 발행물 품질은 지금 **세 겹**으로 막혀 있지만, 세 겹 모두 *내용이 실제로 좋은지*는 측정하지 않는다.

| 기존 게이트 | 무엇을 보나 | 한계 |
|---|---|---|
| v16 §B 슬롯 규칙 (작성 세션) | 복붙 금지·사실격리·과확장 가드를 **지시** | 같은 컨텍스트의 자기 검증 → 같은 실수를 같은 눈으로 놓침(맹점). 예방일 뿐 측정 아님 |
| Publish Lint (작성 세션) | 잔존 토큰·금지 문법·듀얼링크 등 **구조** | 산문 품질 무관 |
| Brief Lint (독립 세션, L1~L10) | 불변식 **PASS/FAIL**(듀얼링크·quote 규율·Status 마감 등 구조) | "해석·요약 품질 평가가 아니라 구조 검사가 1차 목적"(프롬프트 §역할 명시) |

즉 **"카드 79장의 산문이 raw 사실과 정합하는가, 카드마다 차별화됐는가, 시사점·점검이 QA 실무에 실제로 쓸모 있는가"** 를 채점하는 장치가 없다. 2026-06-06 발행본의 P1 복붙·P2 사실 오귀속·P3 제목 절단·P4 다품목 반복은 **사람이 79장을 눈으로 훑어** 발견했고, 그래서 v16 카드내용 패치가 나왔다. 이 발견 과정이 매주 반복되어야 하는데 지금은 **수작업·정성·비반복**이다.

### 1.2 단일 사용자 관점의 함정 (사용자가 제기한 핵심)

현 운영자(QA 1인)는 발행본을 "대충 읽는" 수준으로만 검토하며, **무엇을 봐야 하는지 기준이 없다.** 이는 개인의 문제가 아니라 **단일 관점의 구조적 한계**다. 같은 사람이 매주 같은 방식으로 보면 같은 종류의 결함만(혹은 아무것도) 잡는다.

해결의 방향: 채점자를 **여러 QA 페르소나**로 분화시킨다. 무균·주사제 QA, 바이오/CGT QA, 국내 GMP·실태조사 담당, 원료/DMF 담당 등 서로 다른 관점이 각자 "이 카드가 *나에게* 실행 가능하고 신뢰할 만한가"를 물으면, 단일 운영자가 스킵하던 디테일(제형 추정의 근거 부족, 모달리티 오분류, 점검 항목의 실행 불가능성)이 표면화된다. 이것이 eval 하니스를 단순 회귀 검사가 아니라 **다관점 품질 진단**으로 만드는 설계 축이다.

## 2. 설계 원칙

1. **독립 세션·읽기 전용.** Brief Lint 와 동일하게 작성 세션과 분리된 새 채팅에서 실행한다(맹점 제거). 채점자는 브리프를 **수정하지 않는다** — 점수와 근거만 보고한다(수정은 Lint 트랙의 역할).
2. **근거 대조 채점.** 모든 감점은 "어느 카드의 어느 슬롯이, 어떤 raw/prose_input 사실과 어긋나는가"로 카드·블록 위치를 인용한다. 추측 금지, 확인 불가는 감점 아님(별도 표기).
3. **구조 게이트와 비중복.** L1~L10(구조)은 Brief Lint 가 본다. Eval 은 **내용 품질 차원만** 채점한다(아래 §3). 둘은 이중 게이트.
4. **다관점.** 핵심 차원은 단일 채점이 아니라 **QA 페르소나별 유용성**으로 본다(§3.E).
5. **점수의 추적성.** 결과는 매주 같은 포맷의 스코어카드로 누적 가능해야 한다 → E2(Python 자동화)에서 시계열 회귀 감지의 입력이 된다.
6. **비파괴.** scaffold·collector·golden·v16 프롬프트를 일절 건드리지 않는다. Eval 은 발행물 **위에 얹는** 관측 계층이다(K3 4주 관찰 불훼손).

## 3. 채점 차원 (Eval Rubric E1~E6)

각 차원은 **카드 단위 채점 → 브리프 단위 집계**. 점수는 0~2 (0=결함·2=양호) 또는 위반 카운트로 보고하고, 차원별로 *위반 카드의 doc_id*를 반드시 첨부한다.

### E1. 사실 정합 (Fact fidelity) — 가장 중요
슬롯 산문(TITLE_ISSUE·W1·W5·W6·W7)이 그 카드의 `prose_input`/원문 사실에만 근거하는가.
- E1-a **오귀속**: 타 카드·TL;DR·타 업체의 사실(위반유형·사유·수치)을 차용했는가. (P2 결함 계열)
- E1-b **단정 초과**: 원문에 "미기재/미확인"인 항목을 구체값으로 단정했는가(슬롯 간 모순 포함). (D2 계열)
- E1-c **모달리티/제형 오표기**: 브랜드명만 있는 생물주사제를 화학합성으로 다루는 등(GAP-2), 또는 본문 근거 없이 제형(무균·SC·바이오)을 추정했는가. (과확장 가드 계열)

### E2. 카드 차별화 (Differentiation)
같은 유형(회수·행정처분·ICH·WHO 등) 카드들의 W6/W7 이 **그 카드 고유 사실**을 반영해 서로 다른가, 아니면 유형 공통 일반론을 복붙했는가. (P1 결함 계열) — 동일/유사 문장 클러스터를 찾아 카운트.

### E3. 시사점 품질 (W6 — 해석)
- 해석임이 드러나는 톤인가(사실 단정 아님).
- 과확장 없이 사유에서 **한 단계만** 해석했는가.
- "규제가 이렇게 바뀐다 / 집행 방향" 같은 **방향성 인사이트**를 주는가, 아니면 사실 반복에 그치는가.

### E4. 점검 사항 품질 (W7 — 실행 가능성)
점검 bullet 이 **그 카드 사유에 근거한 실제 확인 가능 항목**인가. 막연한 일반 점검("GMP 준수 확인")이거나 원문 근거 없는 항목이면 감점. → **여기서 페르소나 채점이 가장 효과적**(아래 E6).

### E5. thin 입력 처리 (ICH·WHO·RSS 등 주제명만 있는 카드)
입력이 주제명·요약뿐인 카드에서 원인·위반·조치·날짜를 **지어내지 않고** 적절히 얇게 처리했는가(thin 가드 준수). 과생성 = 감점.

### E6. 다관점 유용성 (Persona usefulness) — 사용자 통찰 반영
각 카드를 아래 페르소나 관점에서 "실행 가능·신뢰 가능"으로 평가. 페르소나가 "이 카드로는 내가 뭘 해야 할지 모르겠다 / 근거가 부족해 못 믿겠다"고 답하면 그 카드는 해당 차원 감점.

| 페르소나 | 관점 |
|---|---|
| 무균·주사제 QA | sterility·CCIT·무균공정 신호의 점검 항목이 구체적인가 |
| 바이오·CGT QA | 모달리티 분류가 맞는가, 바이오 특이 리스크(viral·면역원성)를 반영하는가 |
| 국내 GMP·실태조사 담당 | MFDS 지적사항 excerpt 가 실무 점검으로 연결되는가, 기관 태그 정확한가 |
| 원료·DMF 담당 | 원료/공급망 관련 신호(EDQM·NOC)가 식별·연결되는가 |

> 다관점은 **선택적 심화 모드**다. 매주 기본 채점은 E1~E5 + E6 요약 1개 페르소나, 격주/이상 징후 시 E6 전체 페르소나 패스를 권장(토큰·시간 절약).

## 4. 아키텍처 — 2단계

### E1 단계 (지금): 프롬프트 기반 LLM-as-judge
Brief Lint 와 동일 운영 모델. 발행 직후 독립 새 채팅에 `prompts/GRM_Brief_Eval_실행프롬프트.md` 를 붙여넣고 대상 브리프만 지정 → Notion MCP 로 브리프 + 해당 주 handoff(raw/prose_input 대조용)를 fetch → E1~E6 채점 → 스코어카드 보고.
- **장점**: 코드 0줄, 이번 주 월요일 발행분부터 즉시 적용 가능. K3 관찰에 바로 투입.
- **대조 소스**: 채점의 근거는 handoff v2 의 `prose_input`/`card_scaffold`(LLM 이 무엇을 입력받았는지) + Intake raw. 발행본 산문 vs 입력 사실의 정합을 본다.

### E2 단계 (관찰 후): Python 반자동화
E1 을 N주 돌려 rubric 이 안정되면, 결정론 가능한 부분을 Python 으로 내린다.
- **결정론 영역(Python)**: E2 차별화(W6/W7 문장 유사도 클러스터링 — 임베딩/시퀀스 매칭), 잔존 토큰, 길이/형식, 모달리티 라벨 vs 성분 사전 대조(GAP-2 연계).
- **판단 영역(LLM 잔류)**: E1 오귀속·E3 시사점 품질·E6 페르소나는 LLM 채점 유지.
- **산출**: `brief-eval-{date}.json` 스코어카드 → 시계열 누적 → 회귀 감지(전주 대비 차원별 점수 하락 알림). MON-P2(상태저장 관측성)와 자연 결합.
- **주의**: E2 는 새 채팅 세션이 아니라 발행 파이프라인 인접 코드가 되므로 K3 종료 후 별도 게이트(golden·Codex 검토)를 거친다.

## 5. K3 관찰과의 통합

K3 종료조건은 "4주 연속 Brief Lint 위반 0 + 품질 유지"인데 후자가 정성적이다. Eval 을 붙이면:

- 매주 발행 → ① Brief Lint(구조 PASS/FAIL) + ② Brief Eval(내용 E1~E6 점수) **이중 게이트**.
- K3 종료조건을 **"4주 연속 Lint 위반 0 AND Eval E1(사실정합) 위반 0 AND E2~E5 합산 점수 임계 이상"** 으로 정량화 가능.
- 4주 점수 추세가 우상향/안정이면 객관적 종료 근거, 특정 차원이 반복 위반이면 v16 슬롯 규칙의 추가 패치 트리거.

이렇게 하면 "매주 월요일까지 막연히 대기"가 **"매주 스코어카드를 받아 종료에 다가가는 능동 관찰"**로 바뀐다.

---

## 6. 범위 밖 / 후속

- Eval 은 **발행물을 수정하지 않는다**(Lint 가 수정 담당). 역할 혼선 금지.
- E2 Python 자동화·시계열 회귀·MON-P2 결합은 **K3 종료 후** 트랙.
- GAP-2(브랜드→성분 모달리티) 근본해결은 별도 P0-2 트랙이며, E1-c·E6 바이오 페르소나가 그 결함을 **드러내는** 관측 역할을 한다(수정은 모달리티 트랙).
