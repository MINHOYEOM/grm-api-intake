# GRM Keystone — Codex 디자인 검토 프롬프트 (UX/가독성)

> 사용법: Codex 에 붙여넣는다. 이번 검토는 **구현 타당성이 아니라 디자인/가독성**이다.
> 대상: Notion 견본 페이지(원본 카드 + v2 개선안) + `docs/GRM_card_spec_v16.md` §11~§13.

---

```
[역할] GRM Weekly Brief 카드의 디자인/UX 검토자. 대상 독자는 제약사 QA 실무자이며
매주 반복해서 카드를 스캔·정독한다. 목표 디자인 컨셉은 "전문성 + 스캔성 + 정보 위계"다.
화려함이 아니라 규제/품질 문서다운 차분함·일관성·읽기 편한 구조를 평가한다.

[검토 대상]
1. Notion 견본 페이지(원본 카드 4종 + 하단 "✨ 개선안 미리보기 v2"):
   https://app.notion.com/p/3763142fdc118133b1d3f6a70ae52cf2
2. docs/GRM_card_spec_v16.md  (§11 디자인 미확정, §13 디자인 방향)
3. 참고: docs/prompts/GRM_Prompt_v15.8.md 의 W1~W8 카드 표준

[이미 정해진 사용자 의견(이걸 전제로, 더 나은 안을 제시)]
- 글로벌(FDA 등) 영문 항목은 영문 핵심문장 인용 + 한글 번역을 각각(①② 짝) 제공.
- AI 면책 문구를 처음 또는 마지막에 반드시(문구는 별도 논의).
- 이모지 절제 — 박스 헤더 1개는 OK, 메타표 라벨 이모지·제목 색사각형은 과하다.
- 메타표에서 Evidence·Signal·원문언어 행은 W1 배지와 중복이라 재검토.
- 제품군 명칭은 "화학합성"보다 전문적 표현(합성의약품/바이오의약품 또는 영문).

[검토 질문]
1. 정보 위계: 카드를 3초 스캔할 때 "무슨 일이 / 왜 중요한지"가 먼저 들어오는가?
   W1~W8 순서·강조가 QA 실무 스캔에 최적인가? 더 나은 블록 순서·강조 배치가 있는가?
2. 중복 제거: Evidence/Signal 이 W1 배지와 W2 표에 이중 노출되는 것 외에,
   제목·배지·표·본문에서 같은 정보가 반복되는 지점을 모두 찾고 제거안을 제시.
3. 색·이모지 예산: 카드당 적정 색 callout 수·이모지 수는? 어디는 색이 값을 하고
   어디는 노이즈인가? "점검사항만 강조색" 안의 타당성 평가.
4. 제목 설계: 현재 제목이 길다. 모바일(좁은 폭) 줄바꿈을 고려한 제목 구조 제안
   (유형·기관·핵심대상·DocID 중 무엇을 남기고 무엇을 본문으로 내릴지).
5. 원문/번역 짝: 영문 인용 + 한글 번역의 시각적 결합 방식(①② 번호, 박스 분리/통합 등)
   중 가독성이 가장 좋은 형태는?
6. 일관성: 유형(행정처분·회수·WL·가이던스·실사)이 달라도 같은 골격으로 읽히는가?
   유형별로 흔들리는 지점이 있으면 지적.
7. 접근성·길이: 카드 1장이 한 화면에 들어오는 적정 길이는? 너무 긴 카드를 줄이는
   구체안(예: Raw·긴 표를 toggle 로).
8. v2 개선안 평가: 하단 "✨ 개선안 미리보기 v2" 카드가 원본보다 나아졌는가?
   더 손볼 점은? 과하게 덜어낸 정보는 없는가?

[산출물]
1. 카드 블록 구조·순서·강조에 대한 구체 개선안(현행 ↔ 제안 대비).
2. 색·이모지 예산 권고(카드당 수치·위치).
3. 제목·메타표·원문/번역 짝의 구체 디자인안.
4. v2 개선안 평가 + 추가 수정 제안.
5. 종합: 동결 가능한 "디자인 원칙" 5~8개로 정리(스펙 §13 에 넣을 형태).
※ 코드·발행물 변경 금지. 디자인 제안만.
```
