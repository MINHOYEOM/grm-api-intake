## 4. 단계 계획

### Phase 2 — 저위험 코드 이관 + scaffold 중복 제거 (이번 트랙 우선)

축소가 **약화가 되지 않으려면** 순서가 중요하다: 코드 검사를 먼저 만들고(②), 그 다음 프롬프트의 기계적 항목을 "게이트 실행" 으로 치환한다.

**2-A. ② 기계적 Publish Lint → `brief_lint.py` 순수 함수 (코드 백업)**
- 신규 순수 함수 `lint_publish_structure(published_markdown) -> list[LintFinding]`(brief_lint 패턴 재사용):
  - PL-1: 본문에 `{{` 잔존 0.
  - PL-3/16: `[!NOTE]/[!WARNING]/[!IMPORTANT]/[!TIP]/<toggle>/</toggle>/[toc]/[TOC]` 0 · 빈 callout 0(메타 토글 HARD 포함).
  - PL-10: 카드 H3 제목 bold 구절에 "미상"/"미기재" 0.
  - PL-14: 헤더 메타라인 `{YYYY-MM-DD} ({요일})` 의 요일 == 날짜의 실제 KST 요일.
- CLI 확장: `run_publish_gate` 가 provenance + structure 를 함께 보고(또는 `--structure` 플래그). exit 1 = 차단.
- 테스트: `tests/test_brief_lint.py` 에 구조 lint 회귀 추가(정상/각 위반). **collector·scaffold·golden 불변**(순수 함수·md 입력만) → golden byte-diff 0, 테스트 수 520→증가.

**2-B. ② 대응 프롬프트 축소 (코드가 생긴 뒤)**
- [Publish Lint] 의 기계적 항목(1·3·10·14·16)의 상세 서술을 **"구조 게이트(`brief_lint` structure lint)를 실행해 통과시킨다"** 1~3줄로 치환. 의미 항목(2·5·7·8·9·11·12·13·15·17)은 유지. 17·[발행 전 게이트]는 그대로.
- MCP 전용 세션용 수기 fallback 1줄 유지(항목 17 의 기존 패턴 동일).
- **순 축소 ≈ 12~18줄**, 동시에 자가점검 → 코드 실행으로 **강화**.

**2-C. ③ scaffold 중복·재진술 축약 (순수 프롬프트 삭제, 무약화)**
- [검색 카드 미니 템플릿] L352–353 의 W6/W7 규칙 재진술 → "[2단계] W6/W7 규칙 동일 적용" 참조로 축약(이미 그 표현이 있으므로 중복 문장만 제거).
- [Notion 마크다운 문법] 의 **카드 한정** 설명(scaffold 가 보장)을 줄이고 "scaffold 가 카드 문법을 완성했다(불변). 아래는 네가 새로 쓰는 블록 전용" 을 더 분명히 — 카드 색상/표 형식 재설명 제거.
- **순 축소 ≈ 6~10줄.**

각 편집 단위 후: `python -m unittest`(≥520 green) + golden byte-diff 0(프롬프트만 바꾸므로 자동) + before/after 줄 수 기록.

### Phase 3 — 수집 이관 평가 (①, 더 큰 작업 · 별도 하위 작업으로 분리)

- **Core 8 슬롯 cross-check 축소:** 슬롯 1·3·4·5·6 은 `collect_intake.py`/RSS 가 이미 수집(프롬프트도 "Intake 흡수로 대체" 허용). 이 검색을 "수집기 커버 — 누락 의심 시에만 cross-check" 로 명시 축소하고, **검색 유일경로 슬롯 7(ICH event)·8(TGA)** + graceful 만 최소 유지. → LLM 자유검색 표면 대폭 감소.
- **신규 수집기 후보:** Deep Dive·돌발공지 중 구조화 가능분(NEWSRC-1 EudraGMDP NCR 등)은 `probe_*` 타당성 확인 후 `collect_*.py`(flag off·golden·테스트 동반) **별도 트랙**.
- 못 옮기는 검색 = 슬림 최소 검색 + 명시 graceful fallback(LLM 자유 URL 은 게이트가 계속 검증).
- ⚠️ 행동 변화(커버리지) 동반 → Brief Eval(EVAL-1)·dry-run 으로 정량 비교 후 사람 게이트.

### Phase 4 — 검증 (행동 동등성)

- golden byte-diff 0(의도 외 변경 없음) · 520+ green · URL 게이트/탐지 테스트 통과.
- 동일 입력(최근 handoff)으로 축소 전/후 발행물 카드 구조 비교 — 커버리지·슬롯·링크가 의도된 축소 외 동일. 가능 시 Brief Lint/`GRM_Brief_Eval_harness_design.md` 로 정량.
- before/after 줄·토큰 절감 기록.
